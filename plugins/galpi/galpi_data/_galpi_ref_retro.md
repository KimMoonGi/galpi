# 회고 메모리 3계층 ← Reflexion + Expel + A-MEM

파일: `galpi_data/_galpi_retrospectives.md` (append)

## 3계층 구조

- **에피소드**: 개별 실행 원시 기록 (날짜·모드·기준·패턴·약점·교훈).
- **원칙**: `IF [상황] THEN [원칙] (BECAUSE [근거])` 전이 가능 규칙.
- **관계**: 원칙 간 그래프. `{원칙A} --{강화|충돌|전제}--> {원칙B}`. 교차 도메인 전이 시 탐색.

## 저장 형식 (P3 완료 직후)

```
## {날짜} | {모드} | {의도 1줄} | 경로:{L/S/H}
- 유효기준: {Top 3 기준명}
- 선택패턴: {채택 방안의 공통 접근법 1줄}
- 거부약점: {폐기·거부 방안의 공통 결함 1줄}
- 교훈: {다음에 같은 도메인을 처리한다면 1줄}
- 원칙: IF {상황} THEN {행동} (BECAUSE {근거})
- 관계: {이 원칙} --{강화|충돌|전제}--> {기존 원칙명} (해당 시)
- 골든시드: {Top1 시드 명칭+핵심 1줄} (성공 시만)
- 행동변화: {이전 원칙 → 이번에 실제로 행동을 바꾼 사례. 없으면 "없음, 이유: ~"}
- 미달사항: {달성하지 못한 것 / 타협한 것 1건 이상}
- 프로세스: 시드{}/클러스터{} | 검증:{} | 근거{} | SC{}/{}
- AI예측: {} | 사용자: {미수집|점수} | drift: {}
```

## 참조 (P1 진입 시)

회고 파일 Read → 원칙+관계 계층 우선 로드. **빈도 가중 정렬**: 각 원칙의 관계 엣지(강화·전제·충돌) 중 P1 감지 도메인과 관련된 엣지를 우선 카운트. 최근 3버전 내 엣지에 가산. 엣지 0인 원칙은 P1 감지 도메인 일치 시에만 로드. 강화 원칙 묶음 로드, 충돌 원칙 경고. 골든 시드 → P2 시작점.

## 증류 (원칙 5개 축적 시)

충돌·중복 검사 → 상위 규칙 통합 + 관계 그래프 업데이트. 에피소드는 요약본만 보존.
