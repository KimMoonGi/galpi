# 갈피 품질 측정 프레임워크 v1.0

> 4-Layer 측정 아키텍처. 프로세스 품질과 결과 품질을 분리하되 교차검증으로 자기기만을 방지한다.
> 설계 근거: SDG Decision Quality (6차원, 최약 고리), Brier Score (캘리브레이션), Annie Duke (프로세스≠결과), Morewedge (사전/사후 비교)

## 핵심 원칙

1. **프로세스/결과 분리**: 좋은 프로세스 + 나쁜 결과 = 정당. 나쁜 프로세스 + 좋은 결과 = 경고.
2. **최약 고리**: 종합 점수·평균 금지. 각 Layer, 각 지표를 개별 추적.
3. **적정 범위**: "높을수록 좋다"가 아닌 지표가 있다. 상한/하한 모두 경고.
4. **교차검증**: Layer 간 괴리가 진짜 신호. 단일 Layer만 보면 자기기만.
5. **합산 점수 금지**: Goodhart 방지. 각 지표는 개별로만 기록·비교.

## 구조

```
Layer 4 (결과 캘리브레이션)  ← 분기/반기, 가장 정직, 가장 느림
  ↑ 교차검증
Layer 3 (결정 품질 감사)     ← 월간/분기, 구조화된 외부 시선
  ↑ 교차검증
Layer 2 (지각된 가치)        ← 세션 직후, 사용자 피드백
  ↑ 교차검증
Layer 1 (프로세스 충실도)    ← 매 세션 자동, AI 자기점검
```

## Layer 1: 프로세스 충실도

**시점**: 매 세션 P3 완료 시 자동 기록
**소스**: AI 자기점검 (자기평가이므로 L2-L3와 반드시 교차검증)

### P1 지표 (의도 확정)

| ID | 지표 | 정의 | 측정 | 적정 | 하한 경고 | 상한 경고 |
|:---:|------|------|------|:---:|------|------|
| L1-P1-1 | 리프레임률 | 초기 진술 ≠ 확정 의도 | Y/N per session | 30-60% | <20% P1 형식적 | >70% 억지 리프레임 |
| L1-P1-2 | 3슬롯 구체성 | 대상/조건/결과 검증가능성 | 각 0-2, 합산 0-6 | ≥4 | <3 의도 불명확 진행 | — |
| L1-P1-3 | SC 검증가능성 | SC의 사후 판정 가능성 | SC당 Y/N, 비율 | ≥80% | <60% 측정 불가 SC | — |

### P2 지표 (선택지 확장)

| ID | 지표 | 정의 | 측정 | 적정 | 하한 경고 | 상한 경고 |
|:---:|------|------|------|:---:|------|------|
| L1-P2-1 | 시드 다양성 | Top3 클러스터 출신 다양성 | 3/3, 2/3, 1/3 | ≥2/3 | 1/3 시드 편향 | — |
| L1-P2-2 | 역발상 생존율 | 역발상 시드의 S2 생존 비율 | 비율 | 10-30% | 0% 역발상 무의미 | >50% 기준 느슨 |
| L1-P2-3 | 신규 발견율 | 사용자 미보유 옵션의 Top3 포함 | 비율 | ≥1/3 | 0/3 P2 무가치 | — |

### P3 지표 (결론 검증)

| ID | 지표 | 정의 | 측정 | 적정 | 하한 경고 | 상한 경고 |
|:---:|------|------|------|:---:|------|------|
| L1-P3-1 | 검증 격리도 | 검증자 컨텍스트 미공유 | Y/N | 100% | <100% 격리 실패 | — |
| L1-P3-2 | 반론 구체성 | 반론의 조건 명시 수준 | 반론당 0-2, 평균 | ≥1.5 | <1.0 뭉뚱그린 경고 | — |
| L1-P3-3 | 미달 고백 수 | 타협/미달 기술 건수 | 정수 | ≥1 | 0 과잉자신감 | >5 과잉불안(L만) |
| L1-P3-4 | 루프백률 | 검증 후 수정 비율 | 전체 비율 | 10-30% | 0% 검증 형식적 | >50% 생성 품질 문제 |

## Layer 2: 지각된 가치

**시점**: 세션 직후 (P3.5에서 구조화 수집)
**소스**: 사용자 피드백
**수집**: 기본값 (opt-out 가능, opt-in 아님). 최소 3문항(핵심), 전체 7문항(확장).

### 핵심 3문항 (항상 수집)

| ID | 매핑 | 질문 (사용자 대면) | 척도 |
|:---:|:---:|------|:---:|
| L2-1 | P1 | "고민이 더 선명해졌나요?" | 1-5 |
| L2-2 | P2 | "생각 못한 선택지를 발견했나요?" | Y/N + 개수 |
| L2-3 | P3 | "결론을 반박하기 어려웠나요?" | 1-5 |

### 확장 4문항 (S/H 또는 사용자 동의 시)

| ID | 매핑 | 질문 (사용자 대면) | 척도 |
|:---:|:---:|------|:---:|
| L2-4 | P2 | "최종 선택은 처음부터 있던 것? 새로 발견?" | 기존/신규/혼합 |
| L2-5 | P3 | "제시된 약점이 고려할 가치가 있었나요?" | 1-5 |
| L2-6 | 전체 | "이 결론을 실행할 예정인가요?" | Y/N/부분 |
| L2-7 | 전체 | "의사결정 자신감이 어떻게 변했나요?" | -5~+5 |

### 역방향 경고

| 조건 | 경고 |
|------|------|
| L2-7 자신감 변화 연속 3건+ ≥+4 | "만족도 착각 — 듣고 싶은 말만 해주는 중?" |
| L2-1 명확화 연속 3건+ =5 | "P1 과잉개입 — 사용자 자체가 명확했는데?" |
| L2-2 시야확장 연속 3건+ =N | "P2 무가치 — 시드가 기존 선택지 반복?" |
| L2-6 실행전환 연속 5건+ =N | "실행불가 — 결론이 현실 무시?" |

## Layer 3: 결정 품질 감사

**시점**: 월간/분기, 표본 5-10건 (무작위 또는 L1-L2 괴리건 우선)
**소스**: 인간 외부 평가 (분기 1회 필수) + AI 자가감사 (월간 보조)
**도구**: `_ref_audit_checklist.md`

### SDG 6차원 → 갈피 번역

| 차원 | 원래 | 번역 | 핵심 질문 | 0-100 |
|:---:|:---:|------|------|:---:|
| D1 | Frame | 진짜 문제 | 증상이 아닌 원인을 다뤘는가? 문제 정의가 사용자의 진짜 고민과 일치하는가? | |
| D2 | Alternatives | 선택지 품질 | 누락된 명백한 옵션이 없는가? 창의적이고 충분히 다양한가? | |
| D3 | Information | 근거 충실도 | 핵심 주장에 외부 근거가 있는가? 정보 격차를 명시했는가? | |
| D4 | Values | 가치 반영 | 사용자의 진짜 우선순위/가치관이 기준에 반영되었는가? | |
| D5 | Reasoning | 추론 건전성 | 논리에 비약·순환·확증편향이 없는가? 반론이 실질적인가? | |
| D6 | Commitment | 실행 구체성 | "내일 뭘 하면 되는지" 수준인가? 다음 행동이 명확한가? | |

**전체 점수 = 6개 중 최저값** (SDG "최약 고리" 원칙). 평균 금지.

## Layer 4: 결과 캘리브레이션

**시점**: 결정 후 3개월/6개월
**소스**: 사용자 사후 보고 + 후속 세션 자동 회수
**저장**: `_calibration.md`의 L4 섹션

| ID | 지표 | 정의 | 목표 |
|:---:|------|------|:---:|
| L4-1 | 리스크 캘리브레이션 | "틀릴 수 있는 이유" N개 중 실제 발생 비율 vs 암묵적 확률 | Brier ≤0.25 |
| L4-2 | 후회도 | "다시 한다면 같은 선택?" | ≥3.5/5 |
| L4-3 | AI예측 캘리브레이션 | AI 예측(1-5) vs 사용자 사후 평가(1-5) 절대 격차 | ≤1.0 |
| L4-4 | 골든시드 적중률 | 재사용 시 동일 수준 평가 비율 | ≥70% |

### Annie Duke 안전장치

- L4 결과가 나빠도 L1-L3를 소급 하향하지 않는다.
- 단, L1-L3 고점 + L4 저점이 **3건+ 반복**되면 "체계적 맹점" 조사 트리거.

## 교차검증 매트릭스

| 패턴 | 감지 조건 | 의미 | 행동 |
|------|------|------|------|
| **형식주의** | L1↑ + L2 정체/↓ | 프로토콜 준수하나 실질 가치↓ | P2 시드 품질·P3 반론 깊이 점검 |
| **만족도 착각** | L2↑ + L3↓ | 듣기 좋은 말만 제공 | 적대 검증 강화, 불편한 반론 의무 |
| **자기기만** | L1↑ + L3↓ | 자기평가 과대 | L3 외부 감사 빈도 증가 |
| **체계적 맹점** | L3↑ + L4 3건+ ↓ | 프로세스에 구조적 결함 | 근본 원인 분석, 프로토콜 재설계 검토 |
| **운좋음** | L2↑ + L1↓ | 프로세스 약하나 결과 좋음 | 즉시 L1 강화 (다음에는 운 없을 수 있음) |
| **과잉만족** | L2↑ + L4↓ | 당장 만족하나 결과 나쁨 | L2 자신감 변화 역방향 경고 확인 |
| **Goodhart** | 어떤 Layer든 5건+ 동일 방향 | 측정값 최적화 중 | 전 Layer 교차검증 실행, 측정 방식 재검토 |

## 수집 메커니즘

### L1 (자동)
P3 완료 시 `_measurement_log.md`에 자동 append. 형식은 §기록 형식 참조.

### L2 (기본값 수집)
P3.5에서 핵심 3문항을 기본 제시. 사용자가 "건너뛰기"하면 `미수집` 기록.
목표 수집률: ≥50%. 수집률 <30%가 10건+ 지속 시 수집 방식 재설계 트리거.

### L3 (표본 감사)
**월간**: AI 자가감사 3-5건 (L1-L2 괴리건 우선 선택)
**분기**: 인간 외부 감사 3-5건 (무작위 선택). `_ref_audit_checklist.md` 사용.
인간 감사 없이 3개월 경과 시 경고: "L3 인간 감사 미실행".

### L4 (자동 회수 + 자발적 보고)
**자동 회수**: 후속 세션에서 이전 3개월 내 완료 건 중 L4 미수집 건 확인 → "이전에 다뤘던 [주제]의 결과가 어떻게 되었나요?"
**자발적**: 사용자가 자발적으로 결과 보고 시 즉시 기록.
목표 수집률: ≥20%. (0%보다 무한히 나음)

## 집계 · 추적

### 롤링 윈도우 (최근 20건)
각 지표별 이동 평균/비율. 적정 범위 이탈 시 자동 경고.

### 콜드스타트
최초 20건은 원시 수집만. 적정 범위는 추정치 사용.
20건 도달 후 데이터 기반 적정 범위 재조정.

### 분기 리뷰
L1-L4 전체 요약 생성. 교차검증 패턴 분석. 개선/퇴화 방향 식별.
이전 분기 대비 변화량 기록.
