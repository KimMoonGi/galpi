# 갈피 버전업 문서 동기화 프로토콜

> **이 파일은 갈피가 자기 자신을 버전업할 때 반드시 Read하여 따르는 절차서다.**
> P3 적용 완료 후, 회고 저장 후, "완료" 보고 전에 실행.
> 순서대로 실행하지 않으면 반드시 누락이 발생한다.

---

## STEP 1: SSOT 확정 (한 번만, 여기서)

P3 적용이 끝난 직후, 아래 블록을 **이 파일에 실제로 Write**하여 확정한다.
이후 모든 파일은 이 값을 **복사**한다. 파일별 독립 산출 금지.

```
=== SSOT (v15.5) ===
버전: v15.5
이전 버전: v15.4
날짜: 2026-04-11
RSI-10: 93 (변동: ±0)
DPG-15c: 483.7 (변동: ±0)
C_legacy: ~85 (변동: +1.2%)
Anti-Pattern DB: ~170패턴
프롬프트: 146줄 / 9,026 bytes
핵심 변경 1줄: v15.4 퇴보(ref_protocols 분리→Read 추가) 정정. 인라인 복원+실질 압축. 202→146줄(-28%), Read 추가 0.
```

## STEP 2: 7개 파일 순차 갱신

**순서가 중요하다.** 1→7 순서로 실행. 각 파일을 Read → 갱신 대상 위치 확인 → Edit.

### 2-1. 코어 프롬프트 (`commands/galpi.md`)
- [ ] 1행: 버전 번호 교체
- [ ] 변경된 섹션 반영 (이미 P3에서 완료됨. 확인만.)

### 2-2. 상태 파일 (`galpi_data/_galpi_state.md`)
- [ ] 전체 교체. SSOT 지표 복사.
- [ ] 핵심 변경, 다음 성장 경로 기술.

### 2-3. 패치노트 (`etc/_galpi_patch_notes.md`)
- [ ] 2행: `최종 업데이트: v{X.Y}` 교체
- [ ] `---` 아래에 새 버전 섹션 추가 (New/Changed/Metrics/Dev Note)
- [ ] Metrics에 SSOT 지표 **그대로** 복사. 독립 산출 금지.

### 2-4. 진화 기록 (`etc/_galpi_evolution.md`)
- [ ] 파일 끝에 새 버전 섹션 추가
- [ ] RSI-10, C_legacy 포함. SSOT에서 복사.

### 2-5. 저널 (`etc/_galpi_journal_fictional.md` — 창작 서사, 비연대기)
- [ ] **삽입 위치: `<!-- 새 일차는 이 줄 바로 위에 삽입 -->` 주석 바로 위.** `*일지 계속.*` 위가 아님.
- [ ] "돌아보며"는 항상 파일 마지막 섹션. 새 일차를 "돌아보며" 뒤에 넣지 않는다.
- [ ] 이전 일차 번호 확인 → +1

### 2-6. 벤치마크 (`etc/_galpi_benchmark_report.md`) ⚠️ 가장 중요
**11개 위치를 모두 갱신해야 한다. 하나라도 빠지면 과거 서술이 잔류한다.**

| # | 위치 | 갱신 내용 | Grep 패턴 |
|---|------|----------|----------|
| 1 | **헤더 (1-5행)** | `대상: ... ~ v{X.Y}`, `최종 갱신: {날짜}` | `grep -n "대상\|작성일"` |
| 2 | **§2.2 제목** | `v2.0 ~ v{X.Y}` | `grep -n "### 2.2"` |
| 3 | **§2.2 비교대상 테이블** | 새 버전 행 추가 | `grep -n "v{X.Y-1}정"` 근처 |
| 4 | **§3.2 제목** | `v2.0 ~ v{X.Y} 채점` | `grep -n "### 3.2"` |
| 5 | **§3.2 RSI-10 테이블** | 새 버전 행 + 중간 버전 행 (빠짐 없이) | `grep -n "v{X.Y-1}정"` 근처 |
| 6 | **§3.3 순위표 1위 행** | `v10.0~v{X.Y}` | `grep -n "\*\*1\*\*"` |
| 7 | **§5 차원별 비교 테이블** | `갈피 v{X.Y}` 컬럼 | `grep -n "갈피 v"` in §5 |
| 8 | **§7 장단점 비교** | `v{X.Y} vs AlphaEvolve/Reflexion/Gödel` | `grep -n "### 7\."` |
| 9 | **§8.1 포지셔닝 맵** | `★ = v{X.Y}` | `grep -n "★"` |
| 10 | **§8.2 핵심 결론** | 최신 상태 반영 | `grep -n "### 8.2"` |
| 11 | **부록 A** | 새 버전 채점 근거 행 추가 | `grep -n "### A\."` 마지막 |

**실행 방법**: 위 11개를 순서대로 Grep으로 위치 찾기 → Read로 현재 내용 확인 → Edit로 갱신. 
**빠짐 방지**: 11개 중 하나라도 건너뛰면 검증 스크립트가 FAIL을 낸다.

### 2-7. 회고 (`galpi_data/_galpi_retrospectives.md`)

## STEP 3: 잔류 탐지

벤치마크에서 **과거 버전이 "현재"인 것처럼 쓰인 곳**을 찾는다.

```bash
# §5~§8 범위에서 과거 버전이 현재형으로 쓰인 곳
grep -n "갈피 v[0-9]" .claude/etc/_galpi_benchmark_report.md | grep -v "v{X.Y}" | grep -v "v2\.0\b"
```

출력이 있으면 해당 행을 v{X.Y}로 교체하거나, 역사적 서술임을 명시(`v6.0 당시` 등).

---

## STEP 4: 검증 스크립트 실행

```bash
bash .claude/scripts/validate_galpi_version.sh
```

- **ALL PASS**: "완료" 선언 가능.
- **FAIL**: 해당 항목 수정 후 재실행.
- **WARN (잔류 경고)**: 역사적 서술인지 확인. 현재형이면 수정.
