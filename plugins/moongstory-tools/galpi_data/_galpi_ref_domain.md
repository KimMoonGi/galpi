# 도메인 어댑터 ← GEPA + USP

**감지 (P1):** 요청 텍스트에서 도메인 자동 분류.

| 도메인 | 신호 | 어댑터 행동 |
|:---:|------|------|
| 설계·아키텍처 | 시스템, 구조, 컴포넌트, API, 스키마 | 확장성·유지보수성 기준 가중, 트레이드오프 기준 추가 |
| 전략·정책 | 전략, 방향, 로드맵, 의사결정, 정책 | 실현가능성·이해관계자 기준 가중, 리스크 기준 추가 |
| 제품·기획 | 기획, 기능, 요구사항, 사용자, UX | 사용자 가치·명확성 기준 가중, 검증가능성 기준 추가 |
| 분석·연구 | 논문, 비교, 측정, 데이터, 평가 | 엄밀성·재현성 기준 가중, 근거 기준 추가 |
| 범용 | 매칭 없음 | 기본 기준 유지, 범위 기준 자동 추가 |

**적용:** 감지 도메인의 기준 프리셋을 P1 기준 초안으로 제공. 동일 도메인 골든시드 → P2 주입.
**GEPA 메타 회고 (P3 후):** 어댑터 **부적합 선행 탐색** — 부적합 측면을 먼저 서술 → 반박이 있어야 적합 판정. 반박 불가 시 매핑 수정 제안(회고 기록).

## 커스터마이즈

위 5개 도메인은 기본 프리셋이다. 자신의 업무에 맞게 도메인 행을 추가/수정할 수 있다.
예시: 게임 개발팀이라면 "기획서(GDD) | 기획서, 시스템, 메커닉 | GAME_BIBLE 참조, 6렌즈 기준 자동 삽입"을 추가.
