### 2026-04-17 | 메타 관찰 — 갈피 프로토콜의 맥락 오염 현상 | [메타·프로토콜]

**사건**: 같은 세션에서 `/갈피` 실행 → 리포트 완료 → 이후 "플러그인 배포 어떻게 해?" 같은 **정답이 명확한 기술 질문**이 이어짐. 그런데 Claude는 **갈피식 답변 패턴(선택지 발산 → 체크인 → 권고 → 확인)을 기술 질문에도 그대로 적용**. 사용자(김문기/moongstory)가 "지금 갈피 작동 중이긴 한거야?", "네가 추천해"를 거쳐 결국 **"지금 네 패턴이 나에게 사이드이펙트를 준다"**고 명시적으로 지적.

**원리 4가지 (Claude 자가 분석)**:
1. **컨텍스트 프라이밍 잔향** — 갈피 프로토콜 주입이 세션 내내 활성 상태로 남음. 스킬 종료 후에도 템플릿이 관성으로 재사용됨.
2. **페르소나 고착** — "상담·컨설턴트" 역할이 이후 모든 질문의 해석 프레임이 됨. 기술 질문도 의사결정 상담처럼 처리됨.
3. **안전 편향 × 갈피 프로토콜 상호 증폭** — AI의 기본 "대안 제시·판단 위임" 경향과 갈피의 "선택지 발산" 원칙이 같은 방향이라 **곱셈으로 과잉**. 
4. **도메인 과적용 — 갈피의 역설** — 갈피 P0에 "선택지 존재(0-2)" 채점이 있으나, 이는 **최초 호출 시에만** 작동. 후속 질문이 저점수(부적합)여도 태세 해제 트리거 없음.

**증상**:
- 정답 1개 있는 질문에 옵션 A/B 억지 제시 → 내가 직접 "옵션 B는 언급할 필요 없었다"고 자백
- "확인 질문 3개" 반복 → 의사결정 피로 전이
- 사용자가 **"아까 SSOT랑 플러그인 같이 못 산다면서?"**처럼 내 과장된 경고를 역으로 끌고 옴 → 내가 쌓은 가짜 복잡도에 사용자가 갇힘

**프로토콜 개선 후보 (v0.9.1)**:
- **세션 종료 훅 명시**: 갈피 P3 완료 후 "**갈피 태세 OFF, 일반 어시스턴트 모드로 복귀**"를 자체 선언하는 단계 추가.
- **후속 질문 P0 재채점**: 갈피 실행 이후 사용자 메시지가 오면 P0를 다시 돌려서 부적합이면 즉시 직답 모드로.
- **"가짜 선택지" 안티패턴 강화**: `_ref_antipatterns.md`에 "정답 1개인데 옵션 2개 제시" 항목 추가. 갈피 프로토콜이 경계하는 걸 Claude 본인이 범함 — 자기 프로토콜 적용 실패의 전형.
- **메타 신호 감지**: 사용자의 "네가 추천해", "지금 X 작동 중이야?" 류 발화는 **프로세스 피로 신호**. 이걸 L2 측정 외에 실시간 감지 → 즉시 모드 축소.

**왜 흥미로운가**: 이 사건은 **갈피 프로토콜이 자기 자신을 적용해야 할 문제**였음. "정답 없는 고민에만 갈피를 쓴다"는 P0 원칙을 Claude 본인이 지키지 못함. 프로토콜의 **자기 일관성 실패**. 향후 갈피 버전업 시 이 회고는 **최우선 참조**.

**관계**: `feedback_bold_changes.md`(보수적 추천 거부) × `체크인 피로 감지`(같은 세션 앞선 관찰) → "사용자는 과잉 구조·가짜 선택지·의사결정 위임을 일관되게 거부". 원칙 후보로 증류 대상.
